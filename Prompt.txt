【原壊：コネクトバトル】追加修正依頼（バトル画面＋キャラ選択画面）
※キャラクター情報エリア（左右パネル）はPC/スマホとも現状「完璧」なので、ここには絶対に影響を出さないでください（レイアウト・幅・フォント・余白など全て維持）。

========================
■ バトル画面
========================

①（スマホ）石投下エリアの「石の上が欠ける」問題が未解決
- centerPanel に padding を入れて少し下にずれたが、石の上部が欠ける現象はほぼ改善していない。
- padding で位置調整するのが根本解決になっていない可能性が高いので、原因を切り分けて直してください。

想定原因候補：
- topCanvas（石投下エリア）または親要素が overflow:hidden / clip / transform の影響で描画が上側で切れている
- topCanvas の座標(0,0)起点が想定より上にあり、表示領域外にはみ出している
- boardWrap / centerPanel の transform / scale / translate の影響で topCanvas が実表示領域とズレている
- iOS Safari の viewport / visualViewport 変動で上端が食われている

やってほしいこと（スマホのみ）：
- topCanvas の表示領域が上で欠けないように、「上端のクリップが発生しない」レイアウトにする
  例：topCanvas を boardWrap 内の通常フローではなく、centerPanel 内で確実に確保された領域に固定（高さ確保）する等
- 可能なら、topCanvas.getBoundingClientRect() と実際の石描画Y座標（描画開始点）をログ出して、欠ける理由を確認してから直す

②（PC/スマホ）スライドでハイライト移動した時もSEを鳴らす
現状：
- タップ（クリック）で列ハイライトが変わるときは音が鳴る
- スライド（ドラッグ/スワイプ）で列ハイライトが変わるときは音が鳴らない

要望：
- スライド操作で列が変わった瞬間も、タップ時と同じSEを鳴らす
- ただし「同じ列のまま指が動いている」間は連打しない（列が変わった時だけ1回）

③（スマホ）フィールドが小さい問題が全く改善していない → PCと同じ“サイズ感”に寄せたい
現状：
- スマホ横画面でも中央フィールド（盤面）がPCに比べて明らかに小さい
- 以前の分析どおり、縦基準(vh/baseHeight)がボトルネックで scale が強制的に下がっている可能性が高い
- いまの調整（baseHeightや余白削減）では体感でほぼ変わっていない

要望（スマホのみ）：
- 「PCと同じサイズ感」に近づくよう、スマホでは盤面をもっと大きく表示してほしい
- 具体的には、centerPanel の中で盤面＋石投下エリア＋時間バーが収まる限界まで盤面を拡大する（余白最小）
- そのために、スマホ時は scale 計算の縦基準（baseHeight）をさらに見直すか、そもそも transform:scale 依存を減らしてレイアウトを組み直す（実寸レイアウト）方向も検討してほしい

※重要：
- キャラクター情報エリア（左右パネル）は現状完成しているので、ここを削る/潰す/縮めるのは禁止（盤面拡大のために左右を犠牲にしない）
- 盤面を大きくするための調整は「中央エリア側の設計」で吸収してほしい

■ キャラクター選択画面
========================

④（スマホ）縦持ち時に「横画面にしてください」ラベル表示
要望：
- スマホが縦画面（portrait）の場合、キャラ選択画面に「マッチングしました」と同じデザイン/同じラベルUIで
  『横画面にしてください』
  を表示する（ゲーム操作はできない/推奨しない形でOK）
- 横画面（landscape）にしたら自動で非表示になる

実装ヒント：
- window.matchMedia("(orientation: portrait)") または CSS @media (orientation: portrait) で検知
- iOS Safari対策で resize / orientationchange の両方に追従
