方針：transform: scale() 方式は維持しつつ、
「#lobbyWrap 自体の width/height を scale 後サイズに変更する」のはやめて、
副作用が少ない viewport ラッパー方式で直したい。

要件：
- Firestore / ゲームロジックは触らない
- 1200×730 固定レイアウト（.parent-container）を崩さない
- スマホ横（例：844×390）で「左上だけ見える」問題を解消し、全体が収まること
- marginLeft をJSでいじらない
- #lobbyWrap の width/height をJSでいじらない（内部flexが壊れる可能性があるため）
- 必要ならスマホ時に main の padding-left: 250px を 0 にする（描画可能幅確保）

実装内容：

1) index.html
#lobbyWrap の外側に viewport を追加する：
<div id="lobbyViewport">
  <div id="lobbyWrap"> ...既存... </div>
</div>

2) style.css
viewport側で収める（clip）：
#lobbyViewport {
  width: 100%;
  height: 100%;
  overflow: hidden;
}

#lobbyWrap {
  transform-origin: 0 0;   /* top-left */
  display: inline-block;
}

スマホ時は main の padding-left を無効化：
@media (max-width: 900px){
  main { padding-left: 0; }
}

必要なら #lobbyViewport { height: 100vh; } も検討してほしい。

3) public/scripts/main.js（setupLobbyLayout）
#lobbyWrap には transform のみ適用し、
width/height/margin の操作はしない。
scale は Math.min(1, (vw-16)/1200, (vh-16)/730) でOK。

この方針で差分パッチ（index.html / style.css / main.js）を作ってください。
