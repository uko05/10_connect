【スマホレイアウト最終調整】

前提：
・PC版の左右キャラクター情報エリアは完成済み。絶対に影響を出さないこと。
・スマホのみ修正対象（指定があるもの以外は）。

────────────────────
①（スマホ）縦画面時ラベルを全画面中央に巨大表示

現状：
縦画面時の「横画面にしてください」ラベルが小さい。

要件：
・画面中央固定（position: fixed; inset: 0）
・横幅ギリギリまで大きく（font-size は vw 基準）
・半透明黒背景
・点滅アニメーション付き
・横画面時は完全非表示
・キャラ選択画面のみ適用

────────────────────
②（スマホ）石投下エリアの石が潰れている

現状：
フィールド拡大後、投下エリアの石が縦につぶれている。

推定原因：
canvas の style サイズと内部解像度（width/height）が一致していない。

修正方針：
・transform: scale は使用しない
・topCanvas.width / height を実寸で再設定
・style.width / height も同じ値に統一
・devicePixelRatio を考慮する場合は補正する

────────────────────
③（スマホ）石投下処理が動かない

現状：
フィールドタップしても石が落ちない。

修正要件：
・touchstart / touchend に対応
・clientX は touches[0] から取得
・PC既存の mousedown / mouseup には影響させない
・handleStoneDrop が必ず呼ばれること

────────────────────
④（スマホ）中央フィールドはさらに最大化する

現状：
まだ下に余白があり、PCと同じサイズ感になっていない。

方針：
・スマホでは transform:scale を完全に使用しない
・CSS実レイアウト方式で最大化
・centerPanel の実測高さから
  dropArea + timeBar + padding を差し引いた残りを
  盤面に最大割当
・アスペクト比はPCと同じ 770:800 を維持
・縦基準がボトルネックなら dropArea / timeBar を削減して優先的に盤面を拡大

────────────────────
⑤（PC/スマホ共通）スライド時もSEを鳴らす

要件：
・列が変わった瞬間のみSE再生
・連続同列では鳴らさない
・PCは mousemove 対応
・スマホは touchmove 対応

────────────────────
⑥（スマホ）緑バーの下に余白を追加

要件：
・緑バーの高さと同じ分だけ下に余白を作る
・centerPanel に padding-bottom を付与
・PC版には影響させない

────────────────────

重要：

・左右キャラクター情報エリアは絶対に変更しないこと
・PC版のレイアウトは変更しないこと
・スマホ専用処理は明確に分岐させること
・transform:scale をスマホでは使用しないこと
